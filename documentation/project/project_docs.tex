	\section{Cel projektu}
	Celem projektu jest stworzenie programu w języku C, który służy do automatycznego i estetycznego rozmieszczania
	wierzchołków grafu planarnego w przestrzeni dwuwymiarowej, z wykorzystaniem algorytmu Eadesa lub twierdzenia 
	Tuttego w zależności od wyboru użytkownika. Program działa w trybie nieinteraktywnym/wsadowym.
	\section{Funkcjonalności programu}
	Program oferuje następujące funkcjonalności:
	\begin{enumerate}
		\item Możliwość kontroli sposobu działania programu za pomocą argumentów linii poleceń.
		\item Wczytywanie grafu z pliku tekstowego.
		\item Rozmieszczanie wierzchołków za pomocą algorytmu Eadesa oraz twierdzenia Tuttego.
		\item Zapisywanie informacji o położeniu wierzchołków po ich rozmieszczeniu do pliku tekstowego lub binarnego.
		
	\end{enumerate}
	
	\section{Sposób wykonania programu}
	Schemat wywołania programu wygląda w następujący sposób:
	\begin{mylisting}
./bin/main <plik_wejściowy> <flagi>
	\end{mylisting}
	Na przykład:
	\begin{mylisting}
./bin/main examples/graph_1.txt -t b -o output.bin -s 1234 -a t
	\end{mylisting}
	Powyższa komenda wywoła program w następujący sposób:
	\begin{enumerate}
		\item Program odczyta graf z pliku \texttt{graph\_1.txt}.
		\item Wynik programu zostanie zapisany do pliku binarnego (flaga \texttt{-t}).
		\item Rezultat zostanie zapisany do pliku \texttt{output.bin} (flaga \texttt{-o}).
		\item Program użyje do początkowego rozmieszczenia wierzchołków ziarna \texttt{1234} (flaga \texttt{-s}).
		\item Program użyje twierdzenia Tuttego do rozmieszczenia wierzchołków (flaga \texttt{-a}).
	\end{enumerate}

	\end{mylisting}

	\end{mylisting}
	\subsection{Plik binarny}
	Dane w pliku binarnym zapisywane są w następującej kolejności:
	\begin{enumerate}
		\item Liczba wierzchołków - \texttt{uint32\_t} - $4$ B
		\item Identyfikator wierzchołka - \texttt{uint32\_t} - $4$ B
		\item Współrzędna X wierzchołka - \texttt{float} - $4$ B 
		\item Współrzędna Y wierzchołka - \texttt{float} - $4$ B
	\end{enumerate}
	Kroki 2, 3 i 4 powtarzane są dla każdego wierzchołka grafu. Zapis odbywa się zgodnie z metodą \texttt{Little Endian}
	- zapis od najmniej znaczącego bajtu.

	\section{Sposób działania algorytmów}
	\subsection{Algorytm sprężynowy Eadesa}
	Ten algorytm liczy dla każdego wierzchołka sumę sił odpychania i przyciągania.
	\begin{mylisting}
SpringEmbedder(G, minimalna_sila, maks_iteracji)
//minimalna_sila = bardzo mała wartość ogranicza działanie algorytmu i zatrzymuje pracę
//kiedy maksymalna znaleziona siła jest mniejsza niż minimalna_sila
t = 0
max_sila = duza_wartosc // maksymalna siła działająca na jeden z wierzchołków

while (t < maks_iteracji AND max_sila > minimalna_sila)

    dla każdego wierzchołka v w G
        oblicz siły odpychania od wszystkich innych wierzchołków

    dla każdej krawędzi (u, v) w G
        oblicz siłę przyciągania między u i v

    dla każdego wierzchołka v w G
        zaktualizuj pozycję v na podstawie działającej siły
        zaktualizuj max_sila

    t = t + 1

end while
\end{mylisting}
\textbf{Formuła dla siły odpychania:}
\begin{equation}
	f_{\mathrm{rep}}(p_u,p_v) =
	\frac{c_{\mathrm{rep}}}{\|p_v - p_u\|^2} \cdot \vec{p_u p_v}
\end{equation}
Gdzie:
\begin{itemize}
    \item $p_v, p_u$ -- pozycje wierzchołków $v$ i $u$
    \item $c_{\mathrm{rep}}$ -- stała określająca siłę odpychania
    \item $\vec{p_u p_v}$ -- wektor kierunkowy od $u$ do $v$
    \item $\|\cdot\|$ -- norma euklidesowa wektora
\end{itemize}
\textbf{Formuła dla siły przyciągania:}
\begin{equation}
	f_{\mathrm{attr}}(p_u,p_v) =
	c_{\mathrm{spring}} \cdot
	\log\!\left(\frac{\|p_u - p_v\|}{\ell}\right)
	\cdot \vec{p_u p_v}
\end{equation}
Gdzie:
\begin{itemize}
    \item $p_v, p_u$ -- pozycje wierzchołków $v$ i $u$
    \item $c_{\mathrm{spring}}$ -- stała określająca siłę sprężyny
    \item $\ell$ -- długość optymalna krawędzi
    \item $\vec{p_u p_v}$ -- wektor kierunkowy od $u$ do $v$
    \item $\|\cdot\|$ -- norma euklidesowa wektora
\end{itemize}
\subsection{Twierdzenie Tuttego}
	Ten algorytm działa szukając takiej pozycji dla każdego wierzchołka, aby jego współrzędne były średnią arytmetyczną pozycji jego sąsiadów.
	\begin{mylisting}
TutteEmbedding(G)

wybierz wierzchołki zewnętrznej ściany grafu
ustaw ich pozycje na okręgu

dla każdego pozostałego wierzchołka v
    zapisz równanie: pozycja v = średnia pozycji sąsiadów v

zbuduj układ równań liniowych dla współrzędnych x i y

rozwiąż układ równań

dla każdego wierzchołka v
    ustaw jego pozycję według otrzymanego rozwiązania
	\end{mylisting}

\textbf{Warunek barycentryczny:}
\begin{equation}
	p_v = \frac{1}{\deg(v)} \sum_{u \in N(v)} p_u
\end{equation}
Gdzie:
\begin{itemize}
    \item $p_v$ -- pozycja wierzchołka $v$ na płaszczyźnie
    \item $p_u$ -- pozycja sąsiada $u$ wierzchołka $v$
    \item $N(v)$ -- zbiór sąsiadów wierzchołka $v$
    \item $\deg(v)$ -- stopień wierzchołka $v$ (liczba jego sąsiadów)
    \item $\sum$ -- suma po wszystkich sąsiadach wierzchołka $v$
\end{itemize}
\textbf{Równania liniowe dla współrzędnych:}
\begin{equation}
x_v = \frac{1}{\deg(v)} \sum_{u \in N(v)} x_u
\end{equation}
\begin{equation}
y_v = \frac{1}{\deg(v)} \sum_{u \in N(v)} y_u
\end{equation}
Gdzie:
\begin{itemize}
    \item $x_v, y_v$ -- współrzędne wierzchołka $v$
    \item $x_u, y_u$ -- współrzędne sąsiada $u$ wierzchołka $v$
    \item $N(v)$ -- zbiór sąsiadów wierzchołka $v$
    \item $\deg(v)$ -- stopień wierzchołka $v$
\end{itemize}

\subsubsection{Funkcja \texttt{place\_on\_circle}}
\begin{mylisting}
void place_on_circle(int outer_faces[], Graph *graph, int k, Vector center) {
    // margin to odległość krańca okręga od granicy przestrzeni
    int margin = 20;
    // radius- promień domyślego okręga na którym rozskładamy wierzchołki
    double radius = fmin(graph->width, graph->height) / 2.0 - margin;

    // Iteracja po wierzchołkach zewnętrznego poligonu
    for (int i = 0; i < k; i++) {

        // Kąt pomiędzy kolejnymi wierzchołkami
        double angle = 2.0 * M_PI * i / k;

        // Zmiana pozycji wieszchołków
        graph->nodes[outer_faces[i]].position.x =
        center.x + radius * cos(angle);
        graph->nodes[outer_faces[i]].position.y =
        center.y + radius * sin(angle);
    }

Funkcja odpowiada za geometryczne rozmieszczenie wierzchołków wyznaczonych jako „poligon zewnętrzny” na okręgu. Jest to kluczowy etap przygotowawczy dla algorytmu Tutte'a, ponieważ unieruchomienie tych wierzchołków w formie wypukłego wielokąta gwarantuje poprawne rozmieszczenie pozostałych punktów wewnątrz.

\subsection{Moduł \texttt{eades}}
Moduł ten zawiera implementację algorytmu Eadesa, należącego do grupy algorytmów siłowych (\textit{force-directed graph drawing}). Służy on do automatycznego rozmieszczania wierzchołków grafu w przestrzeni dwuwymiarowej tak, aby krawędzie miały zbliżoną długość, a wierzchołki nie nakładały się na siebie.

\subsubsection{Funkcja \texttt{eades\_algorithm}}
\begin{mylisting}
void eades_algorithm(Graph *graph, 
    double minimum_force, int max_iterations, 
    double ideal_len, double spring_const, int c, double cooling)
{
    int t = 0;
    double max_force_magnitude = 1.0;
    double temperature = 1.0;

    while(t < max_iterations && max_force_magnitude > minimum_force) {
        max_force_magnitude = 0.0;

        // Wyzeruj siły dla bieżącej iteracji
        for(int i = 0; i < graph->num_nodes; i++) {
            graph->nodes[i].force.x = 0;
            graph->nodes[i].force.y = 0;
        }

        // Oblicz siły odpychania między wszystkimi parami wierzchołków
        for(int i = 0; i < graph->num_nodes; i++) {
            for(int j = i + 1; j < graph->num_nodes; j++) {
                compute_repulive(&graph->nodes[i], &graph->nodes[j], c);
            }
        }

        // Oblicz siły przyciągania wzdłuż krawędzi
        for(int i = 0; i < graph->num_edges; i++) {
            compute_attract(graph, &graph->edges[i], spring_const, ideal_len);
        }

        // Zaktualizuj pozycje wierzchołków i znajdź maksymalną siłę
        for(int i = 0; i < graph->num_nodes; i++) {
            Node *node = &graph->nodes[i];
            double current_force_magnitude = count_vector_length(node->force);

            if(current_force_magnitude > max_force_magnitude) {
                max_force_magnitude = current_force_magnitude;
            }

            // Zastosuj siłę do pozycji wierzchołka, skalując przez temperaturę
            Vector displacement = mult_by_num(node->force, temperature);
            node->position = add_vectors(node->position, displacement);
        }

        // Schładzanie
        temperature *= cooling;
        t++;
    }
}
\end{mylisting}

Główna funkcja sterująca procesem układania grafu. W każdej iteracji obliczane są siły odpychania (pomiędzy wszystkimi wierzchołkami) oraz siły przyciągania (tylko wzdłuż krawędzi). Pozycje wierzchołków są aktualizowane o wektor wypadkowej siły pomnożony przez aktualną „temperaturę” układu, która maleje w czasie, co pozwala na stabilizację rysunku.

\textbf{Argumenty:}
\begin{itemize}
    \item \texttt{graph} - wskaźnik na strukturę grafu.
    \item \texttt{minimum\_force} - próg siły, poniżej którego algorytm kończy pracę.
    \item \texttt{max\_iterations} - limit iteracji pętli.
    \item \texttt{ideal\_len} - docelowa długość krawędzi.
    \item \texttt{spring\_const} - stała sprężystości (przyciąganie).
    \item \texttt{c} - stała potrzebna dla obliczenia siły odpychania.
    \item \texttt{cooling} - współczynnik chłodzenia układu (z zakresu 0 do 1).
\end{itemize}

\subsubsection{Funkcja \texttt{tutte\_iteration}}
\begin{mylisting}
int tutte_iteration(Graph *graph,int iteration, int is_fixed[], uint** adj_list, Vector new_pos[], int deg[], Vector center){
    double sum_x = 0.0;
    double sum_y = 0.0;
    // Jeśli wierzchołek należy do poligonu zewnętrznego, zachowujemy jego aktualną pozycję
    if(is_fixed[iteration] == 1) {
        new_pos[iteration] = graph->nodes[iteration].position;
   
        return EXIT_SUCCESS;
    }
        // Specjalna obsługa wierzchołków mających tylko jednego sąsiada
        if(deg[iteration]==1){
             int neighbor = adj_list[iteration][0];
             //Vektor w kietrunku przeciwnym od środku
                    Vector oposite = substract_vectors(graph->nodes[neighbor].position, center);
                    
                    //Dzielimy przez dystancję między centrem a wierzchołkiem żeby otrzymać vektor kirunkowy z długością = 1
                    double len = distance(graph->nodes[neighbor].position, center);
                    if (len != 0) {
                        oposite.x /=len;
                        oposite.y /=len;
                    }
                    // Zabezpieczenie przed dzieleniem przez zero przy zbyt małym dystansie
                    if (len < EPSILON) return EXIT_SUCCESS;

                    //Mnożymy przez ustawioną outer_rad żeby wierzchołek okazał się na odłegłości outer_rad od sąsiada
                    double outer_rad = 10; // nie może być < margin
                    oposite = mult_by_num(oposite, outer_rad);

                    // Ustawiamy nową pozycję jako przesunięcie względem jedynego sąsiada
                    new_pos[iteration].x = graph->nodes[neighbor].position.x + oposite.x;
                    new_pos[iteration].y = graph->nodes[neighbor].position.y + oposite.y;
        }else{
            //Dla wierzchołków o stopniu > 1 stosujemy zasadę środka ciężkości
            for(int j = 0; j <deg[iteration]; j++) {
                    int neighbor = adj_list[iteration][j];
                    sum_x += graph->nodes[neighbor].position.x;
                    sum_y += graph->nodes[neighbor].position.y;
                    
                    }
                    // Jeśli wierzchołek jest izolowany (stopień 0), przerywamy obliczenia
                    if (deg[iteration] == 0) return EXIT_SUCCESS;
                    // Nowa pozycja to średnia arytmetyczna położeń wszystkich sąsiadów
                    new_pos[iteration].x = sum_x/deg[iteration];
                    new_pos[iteration].y = sum_y/deg[iteration];
                }
            
    
    return EXIT_SUCCESS;
}
\end{mylisting}
Funkcja oblicza nową pozycję dla konkretnego wierzchołka. W przypadku wierzchołków o stopniu wyższym niż 1,
 stosowana jest średnia arytmetyczna pozycji sąsiadów. Wierzchołki o stopniu 1 są specjalnie odpychane od środka 
 grafu.

\begin{itemize}
\item \texttt{graph} - wskaźnik na graf.
\item \texttt{iteration} - indeks wierzchołka.
\item \texttt{is\_fixed} - tablica wierzchołków stałych.
\item \texttt{adj\_list} - dynamiczna tablica dwuwymiarowa sąsiadów.
\item \texttt{new\_pos} - tablica, w której zapisywany jest wynik.
\item \texttt{deg} - stopnie wierzchołków.
\item \texttt{center} - geometryczny środek obszaru.
\end{itemize}

\subsubsection{Funkcja \texttt{offset}}
\begin{mylisting}
void offset(Vector new_pos[], int iteration,int jiteration, double difference){
    double angle = ((double)rand() / RAND_MAX) * 2.0 * M_PI;

    double dx = cos(angle) * difference / 2.0;
    double dy = sin(angle) * difference / 2.0;

    new_pos[iteration].x += dx;
    new_pos[iteration].y += dy;

    new_pos[jiteration].x -= dx;
    new_pos[jiteration].y -= dy;
}
\end{mylisting}
Funkcja ta służy do zmiany pozycji dwóch wierzchołków, które znalazły się w tym samym punkcie przestrzeni.
 Rozsuwa je w przeciwnych kierunkach pod losowym kątem o dystans zdefiniowany przez parametr \texttt{difference}.

\textbf{Argumenty:}
\begin{itemize}
\item \texttt{new\_pos} - tablica nowych pozycji.
\item \texttt{iteration}, \texttt{jiteration} - indeksy kolidujących wierzchołków.
\item \texttt{difference} - wartość przesunięcia.
\end{itemize}
