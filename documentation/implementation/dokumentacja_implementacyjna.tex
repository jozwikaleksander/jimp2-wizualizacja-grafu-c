		\item \texttt{graph} - odpowiada za zdefniowanie struktur dla grafu, tworzenie grafu bazując na wczytanym pliku,

		\item \texttt{output} - odpowiada za zapisywanie informacji o pozycji wierzchołków do pliku tekstowego lub binarnego.

\section{Opis poszczególnych modułów}
\subsection{Moduł \texttt{vector}}
Moduł udostepnia funkcję służące do wykonywania operacji na wektorach, a także definiuje ich strukture. Funkcje tego
modułu wykorzystywane są przez moduł \texttt{eades} i \texttt{tuttes}.

\subsubsection{Struktury}
\begin{mylisting}
typedef struct {
    double x;
    double y;
} Vector;
\end{mylisting}
Struktura wektora składa się dwóch liczb zmiennoprzecinkowych dla współrzędnych x i y.
\subsubsection{Funkcja \texttt{add\_vectors}}
\begin{mylisting}
Vector add_vectors (Vector a, Vector b){
    a.x += b.x;
    a.y += b.y;
    return a;
}
\end{mylisting}
Funkcja dodaje do siebie dwa wektory.

\textbf{Argumenty:}
\begin{itemize}
    \item \texttt{a} - pierwszy wektor
    \item \texttt{b} - drugi wektor
\end{itemize}
Funkcja zwraca zmieniony pierwszy wektor.

\subsubsection{Funkcja \texttt{count\_vector\_length}}
\begin{mylisting}
double count_vector_length(Vector a){
    return sqrt(a.x * a.x + a.y * a.y);
}
\end{mylisting}
Funkcja oblicza długość wektora.

\textbf{Argumenty:}
\begin{itemize}
    \item a - wektor
\end{itemize}
Funkcja zwraca długość wektora jako wartość typu \texttt{double}.

\subsubsection{Funkcja \texttt{distance}}
\begin{mylisting}
double distance (Vector a, Vector b){
    double dx = b.x - a.x;
    double dy = b.y - a.y;
    return sqrt(dx * dx + dy * dy);
}
\end{mylisting}
Funkcja oblicza odległość między dwoma wektorami.

\textbf{Argumenty:}
\begin{itemize}
    \item a - pierwszy wektor
    \item b - drugi wektor
\end{itemize}
Funkcja zwraca odległość między wektorami jako wartość typu \texttt{double}.

\subsubsection{Funkcja \texttt{vector\_from\_points}}
\begin{mylisting}
Vector vector_from_points(Vector a, Vector b){
    Vector new;
    new.x = b.x-a.x;
    new.y = b.y-a.y;
    return new;
}
\end{mylisting}
Funkcja tworzy wektor przemieszczenia od punktu reprezentowanego przez wektor \texttt{a} do punktu reprezentowanego przez wektor \texttt{b}.

\textbf{Argumenty:}
\begin{itemize}
    \item a - pierwszy wektor
    \item b - drugi wektor
\end{itemize}
Funkcja zwraca nowy wektor.

\subsubsection{Funkcja \texttt{mult\_by\_num}}
\begin{mylisting}
    Vector mult_by_num(Vector a, double n){
    a.x *=n;
    a.y *=n;
    return a;
}
\end{mylisting}
Funkcja mnoży wektor przez skalar.

\textbf{Argumenty:}
\begin{itemize}
    \item a - wektor
    \item n - wartośc liczbowa typu \texttt{double}
\end{itemize}
Funkcja zwraca nowy wektor, będący wynikiem mnożenia wektora \texttt{a} przez skalar \texttt{n}.

\subsubsection{Funkcja \texttt{negative}}
\begin{mylisting}
Vector negative(Vector a){
    Vector new;
    new.x = -a.x;
    new.y= -a.y;
    return new;
}
\end{mylisting}
Funkcja zmienia znak współrzędnych wektora.

\textbf{Argumenty:}
\begin{itemize}
    \item a - wektor
\end{itemize}
Funkcja zwraca nowy wektor.

\subsection{Moduł \texttt{validation}}
Moduł ten odpowiada za sprawdzenie grafu pod kątem planarności i spójności po jego wczytaniu z pliku. Moduł składa się z
4 funkcji, które zostaną opisane poniżej.

\subsubsection{Funkcja \texttt{is\_graph\_valid}}
\begin{mylisting}
int is_graph_valid(Graph *graph){
    if(is_potentially_planar(graph) == 0)
        return ERR_GRAPH_NOT_PLANAR;
    else if(is_graph_connected(graph) == 0){
        return ERR_GRAPH_NOT_CONNECTED;
    }
    else if(is_graph_planar(graph) == 0){
        return ERR_GRAPH_NOT_PLANAR;
    }
    return 0;
}
\end{mylisting}

Jest to funkcja publiczna używana przez plik \texttt{main.c}, która używa innych funkcji tego modułu w celu weryfikacji grafu
i w razie potrzeby zwrócenia odpowiednich kodów błędu.

\textbf{Argumenty:}
\begin{itemize}
    \item graph - wskaźnik na strukturę grafu.
\end{itemize}
Funkcja zwraca liczbę całkowitą int informującą o rezultacie funkcji.

\subsubsection{Funkcja \texttt{is\_potentially\_planar}}
\begin{mylisting}
int is_potentially_planar(Graph *graph) {
    if (graph->num_nodes < 3) return 1;
    return graph->num_edges <= 3 * graph->num_nodes - 6;
}
\end{mylisting}
Funkcja ta służy to szybkiego sprawdzenia warunku koniecznego planarności grafu. Pozwala to na szybkie odrzucenie grafów,
o których wiadomo że nie są planarne.

\textbf{Argumenty:}
\begin{itemize}
    \item graph - wskaźnik na strukturę grafu.
\end{itemize}
Funkcja zwraca liczbę całkowitą int równą 1 dla grafu planarnego lub o innej wartości dla grafu nieplanarnego.

\subsubsection{Funkcja \texttt{is\_graph\_connected}}
Funkcja ta służy do sprawdzenia czy graf jest spójny za pomocą algorytmu DFS (Depth-first search).

\textbf{Argumenty:}
\begin{itemize}
    \item graph - wskaźnik na strukturę grafu.
\end{itemize}
Funkcja zwraca liczbę całkowitą int równą 1 dla grafu spójnego lub 0 dla niespójnego.

\begin{mylisting}
int is_graph_connected(Graph *graph) {
    if(graph->num_nodes == 0) return 1;

    // Alokacja pamięci
    int *visited = (int*)calloc(graph->num_nodes, sizeof(int));
    int *dfs_res = (int*)malloc(graph->num_nodes * sizeof(int));
    uint **adj_list = (uint**)malloc(graph->num_nodes * sizeof(uint*));
    int *deg = (int*)malloc(graph->num_nodes * sizeof(int));
    int idx = 0;
    int is_connected = 0;
\end{mylisting}
Na początku następuje sprawdzenie ilości wierzchołków grafu oraz alokacja pamięci dla potrzebnych zmiennych.
\begin{mylisting}
    // Sprawdzenie czy udało się zaalokować pamięć
    if (visited == NULL || dfs_res == NULL || adj_list == NULL || deg == NULL) {
        fprintf(stderr, "BŁĄD: Nie udało się zaalokować pamięci.\n");
        if (visited) free(visited);
        if (dfs_res) free(dfs_res);
        if (adj_list) free(adj_list);
        if (deg) free(deg);
        return ERR_MEMORY_ALLOC;
    }
\end{mylisting}
Następnie funkcja sprawdza czy udało się zaalokować pamięć.
\begin{mylisting}
    // Alokacja pamieci dla elementów listy sąsiedztwa
    for (int i = 0; i < graph->num_nodes; i++)
        adj_list[i] = (uint*)malloc(graph->num_nodes * sizeof(uint));
\end{mylisting}
Należy również zaalkować pamięć dla elementów listy sąsiedztwa.
\begin{mylisting}
    // Zbudowanie listy sąsiedztwa
    int result_adj_list = build_adj_list(graph, adj_list, deg);
    
    // Sprawdzenie czy udało się zbudować
    if (result_adj_list != 0){
        fprintf(stderr, "BŁĄD: Nie udało się zbudować listy sąsiedztwa.\n");
    } else {
        // Uruchomienie DFS
        dfs_rec(graph, adj_list, deg, &idx, 0, visited,dfs_res);
        // Jeżeli indeks po wykonaniu DFS jest równy ilości wierzchołków to graf jest spójny
        if(idx == graph->num_nodes)
            is_connected = 1;
    }
\end{mylisting}
W kolejnym kroku tworzona jest lista sąsiedztwa na podstawie grafu. Jeżeli operacja się powiodła to uruchomiony zostaje
algorytm DFS oraz następuje sprawdzenie czy końcowy indeks jest równy ilości wierzchołków co świadczy o spójności grafu.
\begin{mylisting}
    // Zwolnienie pamięci
    free_adj_list(graph, adj_list);
    free_deg(deg);
    free(visited);
    free(dfs_res);

    return is_connected;
\end{mylisting}
Na koniec następuje zwolnienie pamięci oraz zwrócenie zmiennej \texttt{is\_connected}.

\subsubsection{Funkcja \texttt{is\_graph\_planar}}
Funkcja ta wykorzystuje bibliotekę \texttt{planarity} w celu sprawdzenia czy graf jest planarny.

\begin{mylisting}
int is_graph_planar(Graph *graph) {
    graphP gp = gp_New();

    if (gp_InitGraph(gp, graph->num_nodes) != OK) {
        gp_Free(&gp);
        return false; 
    }

    // Dodawanie krawędzi
    gp_AddEdge(gp, index_u, 0, index_v, 0);

    // Uruchomienia funkcji dla sprawdzanie planarności z biblioteki LibPlanarity
    int is_planar = (gp_Embed(gp, EMBEDFLAGS_PLANAR) == OK);

    // Zwalnianie pamięci
    gp_Free(&gp);
    return is_planar;
}
\end{mylisting}

\textbf{Argumenty:}
\begin{itemize}
    \item graph - wskaźnik na strukturę grafu.
\end{itemize}
Funkcja zwraca liczbę całkowitą int równą 1 dla grafu planarnego lub 0 dla nieplanarnego.

\subsubsection{Funkcja \texttt{save\_to\_text}}
\begin{mylisting}
int save_to_txt(Graph *graph, char *output_name) {
    if(!graph) return 1;

    char *final_name = output_name ? output_name : OUTPUT_NAME_TXT;

    FILE *file = fopen(final_name,"w");
    if(!file) return 1;

    for(int i = 0; i < graph->num_nodes; i++) {
        fprintf(file, "%d %lf %lf\n", graph->nodes[i].id, graph->nodes[i].position.x, graph->nodes[i].position.y);
    }
    fclose(file);
    return 0;
}
\end{mylisting}

Funkcja ta odpowiada za wypisywanie informacji o wierzchołkach grafu do pliku tekstowego o wybranej nazwie. Jeżeli nazwa

w dokumentacji funkcjonalnej.

\end{mylisting}

\textbf{Argumenty:}
\begin{itemize}

    \item output\_name - nazwa pliku wyjściowego
\end{itemize}
Funkcja zwraca liczbę całkowitą równą 1 jeżeli operacja się niepowiodła i 0 jeżeli się powiodła.

\subsubsection{Funkcja \texttt{save\_to\_bin}}
\begin{mylisting}
int save_to_bin(Graph *graph, char *output_name) {
    if(!graph) return 1;

    char *final_name = output_name ? output_name : OUTPUT_NAME_BIN;

    FILE *file = fopen(final_name,"wb");
    if(!file) return 1;
    
    uint32_t num_nodes = (uint32_t) graph->num_nodes;
    fwrite(&num_nodes,sizeof(uint32_t),1,file);
    for(int i = 0; i < graph->num_nodes; i++) {
        uint32_t index = (uint32_t) graph->nodes[i].id;
        float x = (float) graph->nodes[i].position.x;
        float y = (float) graph->nodes[i].position.y;

        fwrite(&index,sizeof(uint32_t),1,file);
        fwrite(&x,sizeof(float),1,file);
        fwrite(&y,sizeof(float),1,file);
    }
    fclose(file);
    return 0;
}
\end{mylisting}
Funkcja \texttt{save\_to\_bin} służy do wypisywania danych do pliku binarnego. Podobnie jak dla poprzedniej funkcji, jeżeli
nazwa pliku nie została podana to jest ona ustawiona na domyślną ("output.bin"). Dane są zapisywane w kolejności i typie zdefniowanym
w dokumentacji funkcjonalnej. 

\textbf{Argumenty:}

\end{itemize}
Funkcja zwraca liczbę całkowitą równą 1 jeżeli operacja się niepowiodła i 0 jeżeli się powiodła.

\subsection{Moduł \texttt{graph}}

Moduł ten służy do tworzenia i zarządzania grafami.

\subsubsection{Definicja struktur}
\begin{mylisting}
typedef struct {
    int u, v;
    double weight;
    char *name;
} Edge;

typedef struct {
    Vector position;
    Vector force;
    uint id;
} Node;
\end{mylisting}
Zdefniowane są struktury dla wierzchołków oraz krawędzi grafu. Należy zwrócić szczególną uwagę na fakt, że pole \texttt{id} w wierzchołku nie jest używane w dalszej części programu oprócz w czasie wypisywania do pliku. Jest ono traktowane tylko jako etykieta. Aby operować na wierzchołkach program stosuje indeks tabeli \texttt{nodes} struktury grafu.

\begin{mylisting}
typedef struct{
    Node *nodes;
    int num_nodes;
    int capacity_nodes;
    
    Edge *edges;
    int num_edges;

\subsubsection{Funkcja \texttt{place\_on\_circle}}
\begin{mylisting}
void place_on_circle(int outer_faces[], Graph *graph, int k, Vector center) {
    // margin to odległość krańca okręga od granicy przestrzeni
    int margin = 20;
    // radius- promień domyślego okręga na którym rozskładamy wierzchołki
    double radius = fmin(graph->width, graph->height) / 2.0 - margin;

    // Iteracja po wierzchołkach zewnętrznego poligonu
    for (int i = 0; i < k; i++) {

        // Kąt pomiędzy kolejnymi wierzchołkami
        double angle = 2.0 * M_PI * i / k;

        // Zmiana pozycji wieszchołków
        graph->nodes[outer_faces[i]].position.x =
        center.x + radius * cos(angle);
        graph->nodes[outer_faces[i]].position.y =
        center.y + radius * sin(angle);
    }
}
\end{mylisting}

\subsection{Moduł \texttt{eades}}
Moduł ten zawiera implementację algorytmu Eadesa, należącego do grupy algorytmów siłowych (\textit{force-directed graph drawing}). Służy on do automatycznego rozmieszczania wierzchołków grafu w przestrzeni dwuwymiarowej tak, aby krawędzie miały zbliżoną długość, a wierzchołki nie nakładały się na siebie.

\subsubsection{Funkcja \texttt{eades\_algorithm}}
\begin{mylisting}
void eades_algorithm(Graph *graph, 
    double minimum_force, int max_iterations, 
    double ideal_len, double spring_const, int c, double cooling)
{
    int t = 0;
    double max_force_magnitude = 1.0;
    double temperature = 1.0;

    while(t < max_iterations && max_force_magnitude > minimum_force) {
        max_force_magnitude = 0.0;

        // Wyzeruj siły dla bieżącej iteracji
        for(int i = 0; i < graph->num_nodes; i++) {
            graph->nodes[i].force.x = 0;
            graph->nodes[i].force.y = 0;
        }

        // Oblicz siły odpychania między wszystkimi parami wierzchołków
        for(int i = 0; i < graph->num_nodes; i++) {
            for(int j = i + 1; j < graph->num_nodes; j++) {
                compute_repulive(&graph->nodes[i], &graph->nodes[j], c);
            }
        }

        // Oblicz siły przyciągania wzdłuż krawędzi
        for(int i = 0; i < graph->num_edges; i++) {
            compute_attract(graph, &graph->edges[i], spring_const, ideal_len);
        }

        // Zaktualizuj pozycje wierzchołków i znajdź maksymalną siłę
        for(int i = 0; i < graph->num_nodes; i++) {
            Node *node = &graph->nodes[i];
            double current_force_magnitude = count_vector_length(node->force);

            if(current_force_magnitude > max_force_magnitude) {
                max_force_magnitude = current_force_magnitude;
            }

            // Zastosuj siłę do pozycji wierzchołka, skalując przez temperaturę
            Vector displacement = mult_by_num(node->force, temperature);
            node->position = add_vectors(node->position, displacement);
        }

        // Schładzanie
        temperature *= cooling;
        t++;
    }
}
\end{mylisting}

Główna funkcja sterująca procesem układania grafu. W każdej iteracji obliczane są siły odpychania (pomiędzy wszystkimi wierzchołkami) oraz siły przyciągania (tylko wzdłuż krawędzi). Pozycje wierzchołków są aktualizowane o wektor wypadkowej siły pomnożony przez aktualną „temperaturę” układu, która maleje w czasie, co pozwala na stabilizację rysunku.

\textbf{Argumenty:}
\begin{itemize}
    \item \texttt{graph} - wskaźnik na strukturę grafu.
    \item \texttt{minimum\_force} - próg siły, poniżej którego algorytm kończy pracę.
    \item \texttt{max\_iterations} - limit iteracji pętli.
    \item \texttt{ideal\_len} - docelowa długość krawędzi.
    \item \texttt{spring\_const} - stała sprężystości (przyciąganie).
    \item \texttt{c} - stała potrzebna dla obliczenia siły odpychania.
    \item \texttt{cooling} - współczynnik chłodzenia układu (z zakresu 0 do 1).
\end{itemize}

\subsubsection{Funkcja \texttt{compute\_repulsive}}
\begin{mylisting}
void compute_repulsive(Node *u, Node *v, int c) {
    Vector repulsive; //vector siły odpychania
    double d = distance(u->position, v->position);
    if(d < 0.0001) { //zapobieganie dzieleniu przez zero
        d = 0.0001;
    }
    double rep_num_part = c / (d * d); //część numeryczna siły odpychania
    Vector direction_uv = vector_from_points(u->position, v->position); // Wektor v - u
    Vector unit_vec; // Wektor jednostkowy w kierunku od u do v
    unit_vec.x = direction_uv.x / d;
    unit_vec.y = direction_uv.y / d;
    repulsive = mult_by_num(unit_vec, rep_num_part); //końcowa wartość siły odpychania

    // Siła odpychająca na 'u' działa w kierunku przeciwnym do 'v'
    u->force = add_vectors(u->force, negative(repulsive));
    // Siła odpychająca na 'v' działa w kierunku przeciwnym do 'u'
    v->force = add_vectors(v->force, repulsive);
}
\end{mylisting}
Funkcja oblicza siłę odpychania działającą między dwoma wierzchołkami zgodnie z prawem odwrotnych kwadratów. Siła ta zapobiega nakładaniu się wierzchołków na siebie.

\textbf{Argumenty:}
\begin{itemize}
    \item \texttt{u}, \texttt{v} - wskaźniki na wierzchołki, między którymi liczone jest odpychanie.
    \item \texttt{c} - stała określająca moc siły odpychającej.
\end{itemize}

\subsubsection{Funkcja \texttt{compute\_attract}}
\begin{mylisting}
void compute_attract(Graph *graph, Edge *e, double spring_const, double ideal_len) {
    Node *u = &graph->nodes[e->u]; //2 wierzchołki nałeżające do tej krawiędzi
    Node *v = &graph->nodes[e->v];

    Vector attractive; // vector siły przyciągania

    double d = distance(u->position, v->position);
    if(d < 0.0001) { //zapobieganie dzieleniu przez zero
        d = 0.0001;
    }
    
    Vector direction_uv = vector_from_points(u->position, v->position); // Wektor v - u
    Vector unit_vec; // Wektor jednostkowy w kierunku od u do v
    unit_vec.x = direction_uv.x / d;
    unit_vec.y = direction_uv.y / d;
    double attr_num_part = spring_const * log(d / ideal_len); // numeryczna część siły przyciągania
    attractive = mult_by_num(unit_vec, attr_num_part);  //końcowa wartość siły przyciągania
    // Siła przyciągająca na 'u' działa w kierunku 'v'
    u->force = add_vectors(u->force, attractive);
    // Siła przyciągająca na 'v' działa w kierunku 'u'
    v->force = add_vectors(v->force, negative(attractive));
}
\end{mylisting}
Funkcja oblicza siłę przyciągania wzdłuż krawędzi grafu. Wykorzystuje model logarytmiczny sprężyny, który dąży do ustawienia wierzchołków w odległości równej \texttt{ideal\_len}.

\textbf{Argumenty:}
\begin{itemize}
    \item \texttt{graph} - wskaźnik na graf (do pobrania struktur wierzchołków).
    \item \texttt{e} - wskaźnik na krawędź, wzdłuż której działa siła.
    \item \texttt{spring\_const} - sztywność „sprężyny”.
    \item \texttt{ideal\_len} - pożądana długość krawędzi.
\end{itemize}
