	Program został podzielony na 6 modułów z których każdy składa się z pliku C i pliku nagłówkowego. Moduły to:
	\begin{itemize}
        \item \text {input} - odpowiada za wczytywnia parametrow z linii polecen.

		\item \texttt{graph} - odpowiada za zdefniowanie struktur dla grafu, tworzenie grafu bazując na wczytanym pliku, 
		zarządzanie pamięcią zaalokowaną dla struktur grafu.
		\item \texttt{validation} - odpowiada za sprawdzenie czy graf jest spójny i planarny.
		\item \texttt{eades} - odpowiada za uruchomienie funkcji implementującej algorytm Eadesa.
		\item \texttt{tuttes} - odpowiada za uruchomienie funkcji iplementującej twierdzenie Tuttego.
		\item \texttt{vector} - odpowiada za wykonywanie operacji na wektorach.
		\item \texttt{output} - odpowiada za zapisywanie informacji o pozycji wierzchołków do pliku tekstowego lub binarnego.
	\end{itemize}

	\section{Opis głównego pliku}
	Plik \texttt{main.c} zarządza przebiegiem programu oraz odpowiada za wczytywanie parametrów wykonania programu z linii poleceń.


\begin{mylisting}
int main(int argc, char **argv) {
    // Pobranie parametrów wykonania programu z linii poleceń
    Parameters params = create_params();
    Jeśli load_params(argc,argv,&params) nie działa:
        return EXIT_FAILURE

    Jeśłi nie udało się otworayć
        return ERR_FILE_OPEN;
    

    // Sprawdzenie czy ziarno zostało podano jako argument linii poleceń
    if(params.wasSeedProvided == 1)
        srand(params.seed);
    else
        srand(time(NULL));

    // Stworzenie grafu na podstawie pliku
    int load_graph_status = 0;
    Graph *graph = load_graph(params.graph_file, params.width, params.height, &load_graph_status);

    Jeśli nie udało się wczytać graf
       return ERR_GRAPH_LOAD;
    
    Jeśli nie udała się alokacja pamięci
        return ERR_MEMORY_ALLOC;
    }

    Sprawdzenie czy graf jest spóny, planarny, i alokację pamięci,
    jeśli nie, to zwracanie odpowiednego kodu błędu  
    
    // Obsluga wyboru algorytmu z linii polecen
    Jeśli wybrany Eades
            eades_algorithm(graph, params.minimum_force, params.max_iterations, params.ideal_len, params.spring_const, 
                params.repulsion_const, params.cooling);
            
    Jeśli wybrany Tuttes
            tuttes_algorithm(graph, params.max_iterations);
            
    Domyślnie
        eades_algorithm(graph, params.minimum_force, params.max_iterations, params.ideal_len, params.spring_const, params.repulsion_const, params.cooling);

    // Obsluga zapisu do wybranego typu pliku
    Jeśli nie udało się zapisać do pliku tekstowego
                return ERR_SAVE_TEXT_FAILED;
    Jeśli nie udało się zapisac do pliku binarnego
                return ERR_SAVE_BINARY_FAILED;
  
    free(params.output_name);
    fclose(params.graph_file);

    printf("Pomyślnie zapisano informacje o %d wierzcholkach do pliku.\n", graph->num_nodes);
    free_graph(graph);

    return EXIT_SUCCESS;
}
\end{mylisting}
Na koniec program zwalnia pamięć, zamyka pliki i wyświetla komunikat o pomyślnym zapisaniu danych.

\subsubsection{Funkcja \texttt{save\_to\_bin}}

int save_to_bin(Graph *graph, char *output_name) {
    if(!graph) return 1;

    char *final_name = output_name ? output_name : OUTPUT_NAME_BIN;

    FILE *file = fopen(final_name,"wb");
    if(!file) return 1;
    
    uint32_t num_nodes = (uint32_t) graph->num_nodes;
    fwrite(&num_nodes,sizeof(uint32_t),1,file);
    for(int i = 0; i < graph->num_nodes; i++) {
        uint32_t index = (uint32_t) graph->nodes[i].id;
        float x = (float) graph->nodes[i].position.x;
        float y = (float) graph->nodes[i].position.y;

        fwrite(&index,sizeof(uint32_t),1,file);
        fwrite(&x,sizeof(float),1,file);
        fwrite(&y,sizeof(float),1,file);
    }
    fclose(file);
    return 0;
}

void free_graph(Graph *graph) {
    if (graph) {
        for (int i = 0; i < graph->num_edges; i++) {
            free(graph->edges[i].name);
        }

        free(graph->edges);
        free(graph->nodes);
        free(graph);

\subsubsection{Funkcja \texttt{build\_adj\_list}}
\begin{mylisting}
int build_adj_list(Graph *graph, uint **adj_list, int *deg) {
    int n_nodes = graph->num_nodes;
    int n_edges = graph->num_edges;

    // Zapełnienie deg
    for (int i = 0; i < n_nodes; i++)
        deg[i] = 0;

    // Zapełnienie adj_list
    for (int i = 0; i < n_edges; i++) {
        int u = graph->edges[i].u;
        int v = graph->edges[i].v;

        adj_list[u][deg[u]++] = v;
        adj_list[v][deg[v]++] = u;
    }
    return EXIT_SUCCESS;
}
\end{mylisting}
Funkcja ta przekształca listową reprezentację krawędzi w listę sąsiedztwa, co jest używane przez algorytm tuttesa dla efektywnego poszukiwania sąsiadów.
 W pierwszym kroku zeruje tablicę stopni wierzchołków, a następnie iteruje po krawędziach, dodając sąsiadów do odpowiednich list.

\textbf{Argumenty:}
\begin{itemize}
    \item \texttt{graph} - wskaźnik na strukturę grafu.
    \item \texttt{adj\_list} - tablica dwuwymiarowa przechowująca sąsiadów wierzchołków.
    \item \texttt{deg} - tablica przechowująca liczbę sąsiadów dla każdego wierzchołka.
\end{itemize}
Funkcja zwraca \texttt{EXIT\_SUCCESS} po poprawnym zbudowaniu struktury.

\subsubsection{Funkcja \texttt{find\_outer\_face}}

Funkcja ta stanowi punkt wejścia do algorytmu wyznaczania zewnętrznego poligonu. Jej zadaniem jest przygotowanie struktur pomocniczych (tablicy odwiedzin oraz tablicy przodków) i uruchomienie dedykowanego algorytmu przeszukiwania w głąb (\texttt{dfs\_rec\_face}), który zidentyfikuje pierwszy napotkany cykl.

\textbf{Argumenty:}

\subsection{Moduł \texttt{eades}}
Moduł ten zawiera implementację algorytmu Eadesa, należącego do grupy algorytmów siłowych (\textit{force-directed graph drawing}). Służy on do automatycznego rozmieszczania wierzchołków grafu w przestrzeni dwuwymiarowej tak, aby krawędzie miały zbliżoną długość, a wierzchołki nie nakładały się na siebie.
